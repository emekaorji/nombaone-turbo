Subscription kan ń fi **customer** kan sí orí **price** kan, ó sì ń kówó rẹ̀ ní
cycle kọ̀ọ̀kan. Subscription resource kò kàn ún rail kan pàtó: `POST /v1/subscriptions`
kan náà ni ó ń bẹ̀rẹ̀ subscription card, subscription direct debit, tàbí subscription
bank transfer. Ohun tí ó yàtọ̀ ni **payment method** tí o so mọ́ ọn, nítorí pé àwọn
rail ń gba owó ní ọ̀nà tí ó yàtọ̀ síra: card àti mandate ni a ń *fà* wọlé, bank
transfer ni a ń *tì* wá. Wo
[multi-rail: push àti pull](/concepts/multi-rail-push-and-pull) fún ìdí tí ìyàtọ̀
yìí fi ṣe pàtàkì.

## Kò tíì sí payment method? Hosted checkout

Bọ́tìnì "subscribe" orí storefront kì í sábà ní ohun èlò tí a ti fọwọ́ sí. Fún
ọ̀ràn yẹn, **fo `paymentMethodId` sílẹ̀ pátápátá** lórí ìṣẹ̀dá `charge_automatically`
tí kò ní àkókò ìdánwò:

```bash
curl -X POST https://sandbox.api.nombaone.xyz/v1/subscriptions \
  -H "Authorization: Bearer nbo_sandbox_…" \
  -H "Content-Type: application/json" \
  -H "Idempotency-Key: $(uuidgen)" \
  -d '{
    "customerId": "{customerId}",
    "priceId": "{priceId}",
    "callbackUrl": "https://yourapp.com/billing/return"
  }'
```

Subscription náà bẹ̀rẹ̀ ní `incomplete`, response náà sì gbé **`checkoutLink`**
wá — ojú-ìwé ìsanwó tí Nomba ń gbàlejò (card tàbí bank transfer). Rán customer
náà lọ síbẹ̀; `callbackUrl` àṣàyàn ni ibi tí wọn yóò padà sí lẹ́yìn ìsanwó.
Sísanwó mú subscription náà ṣiṣẹ́; ìsanwó **card** tún gba token tí a lè tún lò,
kí àwọn renewal lè gba owó ní ìdákẹ́jẹ́, nígbà tí ìsanwó **transfer** (tí kò fi
token kankan sílẹ̀ láti fà) yí subscription náà padà sí `send_invoice`: cycle
kọ̀ọ̀kan tí ó ń bọ̀ yóò gbé invoice jáde pẹ̀lú ọ̀nà ìsanwó dípò fífà. Tí ẹnikẹ́ni
kò bá san, subscription náà yóò kọjá sí `incomplete_expired`, kò sì sí gbèsè
kankan. [Recipe gym membership](/cookbook/bill-a-gym-membership) ń lo ẹnu-ọ̀nà
yìí láti ìbẹ̀rẹ̀ dé òpin.

Ṣé o ti ní method tí a ti fọwọ́ sí tẹ́lẹ̀? Ṣàgbékalẹ̀ ọ̀kan kí o sì fi í ránṣẹ́ ní
tààràtà:

## Ṣàgbékalẹ̀ payment method

Àwọn pull rail nílò ohun èlò tí a ti fọwọ́ sí, tí ó sì wà nínú àkọsílẹ̀. Bẹ̀rẹ̀ setup
kan: customer yóò fọwọ́ sí i (fún card, ó sì lè ní láti parí ìgbésẹ̀ OTP bánkì) ní
ọ̀nà ìjápọ̀ tí a dá padà:

```bash
curl -X POST https://sandbox.api.nombaone.xyz/v1/payment-methods/setup \
  -H "Authorization: Bearer nbo_sandbox_…" \
  -H "Content-Type: application/json" \
  -d '{
    "customerRef": "{customerId}",
    "amountInKobo": 250000,
    "callbackUrl": "https://yourapp.com/billing/return"
  }'
```

A ó rán customer náà lọ láti parí ìfọwọ́sí, lẹ́yìn náà a ó dá a padà sí
`callbackUrl` rẹ. Payment method tí ó jáde níbẹ̀ ni ohun tí o máa gbà owó lé lórí ní
cycle kọ̀ọ̀kan.

Fún subscription **bank transfer**, dípò bẹ́ẹ̀, ṣẹ̀dá virtual account kan: customer
yóò máa tì owó sí i ní ọjọ́ ìdíyelé kọ̀ọ̀kan:

```bash
curl -X POST https://sandbox.api.nombaone.xyz/v1/payment-methods/virtual-account \
  -H "Authorization: Bearer nbo_sandbox_…" \
  -H "Content-Type: application/json" \
  -d '{ "customerRef": "{customerId}", "expectedAmount": 250000 }'
```

<Callout type="note" title="Ṣe ìdánrawò tí ìyọrísí rẹ̀ dájú nínú sandbox">
  Nínú sandbox, fo ìgbésẹ̀ ojú ìwé tí a gbàlejò: `POST /v1/sandbox/payment-methods`
  pẹ̀lú `behavior` kan (`success`, `requires_otp`, `decline_insufficient_funds`)
  yóò fún ọ ní method kan tí ìyọrísí rẹ̀ ti dájú, kí o lè ṣe ìdánrawò ọ̀nà tí ó dára
  àti èyí tí kò dára kí owó gidi tó wọlé.
</Callout>

## Bẹ̀rẹ̀ subscription náà

Fi customer náà sí price náà pẹ̀lú method náà:

```bash
curl -X POST https://sandbox.api.nombaone.xyz/v1/subscriptions \
  -H "Authorization: Bearer nbo_sandbox_…" \
  -H "Content-Type: application/json" \
  -H "Idempotency-Key: $(uuidgen)" \
  -d '{
    "customerId": "{customerId}",
    "priceId": "{priceId}",
    "paymentMethodId": "{paymentMethodId}",
    "collectionMethod": "charge_automatically"
  }'
```

`status` tí ó dá padà yóò sọ ibi tí cycle àkọ́kọ́ dé fún ọ:

- **`active`**: a ti gba invoice àkọ́kọ́. O ti ń kówó.
- **`trialing`**: price náà ní àkókò ìdánwò; kò tíì sí charge kankan.
- **`past_due`**: charge àkọ́kọ́ kùnà; [dunning](/guides/dunning-and-recovery)
  ló ni iṣẹ́ ìgbàpadà báyìí. Lórí balance tí kò lágbára, èyí sábà máa ń túmọ̀ sí "kò tíì ṣeé ṣe", kì í ṣe "kò ṣeé ṣe rárá".
- **`incomplete`**: customer ṣì ní láti ṣe nǹkan kan — yálà o lo ẹnu-ọ̀nà hosted
  checkout (rán wọn lọ sí `checkoutLink` inú response), tàbí charge card kan bá
  ìgbésẹ̀ OTP pàdé, event `invoice.action_required` yóò sì gbé ọ̀nà ìjápọ̀
  checkout tuntun wá.

<Callout type="warning" title="Má ṣe rò pé charge àkọ́kọ́ yóò yege">
  Ní Nàìjíríà, pull àkọ́kọ́ lè nílò OTP, tàbí kí ó bá balance tí kò lágbára pàdé. Ṣe
  ìtọ́jú `past_due` àti `incomplete` láti ọjọ́ kìíní: pín ọ̀nà rẹ lórí webhook, kì í
  ṣe lórí `201` nìkan. Èyí ni ìyàtọ̀ láàrin ohun ìṣeré àti ètò ìdíyelé gidi.
</Callout>

## Àṣàyàn: àkókò ìdánwò àti quantity

- **`trialDays`** nínú ìpè ìṣẹ̀dá ń bẹ̀rẹ̀ àkókò ìdánwò kan, kódà bí price náà kò bá ní ọ̀kan.
- **`quantity`** ń bí price orí ẹnì kọ̀ọ̀kan ní ìlọ́po.
- **`Idempotency-Key`** jẹ́ kí ìṣẹ̀dá náà wà ní ààbò fún ìgbìyànjú léraléra: ìpè
  tuntun pẹ̀lú key kan náà yóò dá subscription kan náà padà, kì í ṣe èkejì.
