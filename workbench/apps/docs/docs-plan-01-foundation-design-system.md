# Docs — Plan 01 · Foundation & design-system polish

> Bring the existing chrome + island kit to token-grade and finish the shell so every later phase builds on a polished, accessible, on-brand surface. Prerequisites: 00 (Overview & contract). Serves tenets: 2, 4, 6, 7, 9, 10.

## Goal

The docs already ship a high-craft engine (16 chrome components, ~19 MDX island/primitive files, 7 signature experiences, a live playground proxy) but it is painted in the **wrong brand**. The docs currently inherit the **purple** admin design language from `@nombaone/ui/globals.css` (`--purple-*` ramp, `#695ab0` primary, dark bg `#121518`, dark mode "dormant"), while the website/console brand — the one the docs must match — is the **emerald** language sourced 1:1 from `NOMBAONE.pen` (`--accent #0bdfa3 / #00a473`, background `#040404`, Geist, the 8px radii scale). There are **76 hard-coded `purple` references across 23 docs files** plus a purple Shiki theme and a stale "purple ramp" comment.

Phase 01 does five things and no more: (1) migrate the docs off the purple inheritance onto the emerald `NOMBAONE.pen` token layer so the accent matches website/console everywhere; (2) finish the shell (a reading-measure content column via `--doc-shell-max`, a real three-column layout where the reference needs it, the Geist type scale); (3) make dark the default with a genuine light theme at parity; (4) pass the §4 accessibility gate as a hard gate; (5) polish the home shell, the 404/loading/empty states, and publish a component **inventory** table that hands every island to the phase that will de-example it.

**Out of scope (do not do here):** authoring content pages, writing `manifest.ts` IA (→ 02), the runnable spine / snippet engine / honesty gate (→ 03), new islands like `<ErrorExplorer>`/`<LifecycleSimulator>` (→ 04–08). This phase touches **presentation only**; it must not change any island's data contract or props.

## Antipattern watch (the §10 rules most relevant here)

- **Dark mode without contrast math** — offering a theme ≠ passing WCAG. The re-toned Shiki + emerald focus rings must be *measured* to ~7:1 / ≥3:1, not eyeballed. This is the phase's hard gate.
- **Empty states / errors that dead-end** — 404, loading, and ⌘K-empty must each name the next action (tenet 9 in miniature); no bare spinner, no dead "Back" as the only path.
- **Two brands, one product** — shipping docs in admin-purple while website/console are emerald violates tenet 10 ("one platform, one language"). The migration is the whole point; a single stray `text-purple-*` fails the phase.
- **Presets frozen on the throwaway `example` resource** — not fixed here (that's 04–08), but the inventory must *flag* every signature island still bound to `example` so the debt is visible and owned, not silently shipped.
- **Marketing voice on the docs front door** — the home shell leads with the developer's goal + the positioning line, not benefit-selling adjectives.
- **Scope creep into later phases** — no `manifest.ts` IA (02), no proxy/snippet/honesty work (03), no new islands (04–08). Presentation only; do not alter any island's data contract.

## Manifesto ties

- **Tenet 10 (one platform, one language)** — the accent migration makes docs, website, and console literally one brand; the inventory + token layer are the mechanism that keeps them from drifting.
- **Tenet 2 & 4 (developer-first; curiosity → relief)** — a polished, fast, accessible shell with a real reading measure and a dark-default surface is the "2am relief" substrate every later runnable experience sits on.
- **Tenet 9 (errors are a feature)** — the 404/loading/empty states model the "always offer the next action" discipline the `/errors` museum will scale up in 04.
- **Tenet 6 & 7 (docs are the demo; merchants deserve the same care)** — one design system + the developer/merchant `sidebar-view-toggle` polished here means the no-code track (09) inherits parity for free.
