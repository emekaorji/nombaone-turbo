# Docs — Plan 06 · Get started (Tutorials) + the buffet quickstart grid
> One-line goal: take an anonymous reader from the landing page to a **real `201` first subscription in under 10 minutes, in their own stack**, with zero signup. Prerequisites: 03 (the runnable spine). Serves tenets: 2 (developer-first), 3 (a buffet), 4 (curiosity → first success in minutes), 6 (docs are the demo), 8 (honest about the hard parts).

## Goal

Ship the **Get started** section as pure Diátaxis **Tutorials** — learning-oriented, command/outcome-first, no reference tables. The section must deliver five felt outcomes:

1. **Choice-first onboarding.** A landing grid of icon cards — Node, Next.js, Python, Go, PHP, Ruby, plus **CLI** and **curl** — each a *self-contained copy-run path* to a real `201` in the reader's own language. No monolithic "getting started" in a stack they don't use.
2. **A `<10-minute` "Your first subscription" tutorial** with **progressive disclosure**: one minimal working call up top, edge cases collapsed. It reaches a genuine `201` against the real sandbox via the **ephemeral test key** (Phase 03), so a first-timer never signs up, never copies a key.
3. **Authentication + Environments & keys** pages that state the `nbo_test_`/`nbo_live_` prefix rule and the **deployment-pin reality** (test base `sandbox.api.nombaone.xyz` vs live `api.nombaone.xyz`) in launch-first voice, no endpoint reference.
4. **"Verify us in your own devtools"** — a first-run step where the reader registers a webhook, fires `/v1/test/webhooks/simulate` to emit a **real signed** `invoice.paid` / `invoice.action_required`, and watches it land + pass signature verification in their own tunnel/logs. This is the honest substitute for a sandbox that sends no webhooks.
5. **A maintained clone-and-run starter repo** pointer, and — **optional/later** — a StackBlitz WebContainer zero-install terminal that is *explicit* it cannot receive webhooks in-browser (paired with simulate).

Voice throughout: **Fly.io launch-first** — lead every page with the command and the outcome, defer architecture, cut marketing language. Diátaxis discipline: these are Tutorials; anything that wants to be a field table or an exhaustive parameter list belongs in **Phase 08 (API Reference)** and is *linked*, never inlined.

## Prerequisites (what must exist first)

- **Phase 03 — the runnable spine (hard dependency).** Specifically:
  - **The ephemeral sandbox-key route** (`apps/docs/src/app/api/sandbox-key/route.ts` per §7) that mints a scoped, short-lived, rate-limited `nbo_test_` key + throwaway org and auto-populates `<ApiExplorer>`'s `localStorage` key. Without it, "zero-signup real `201`" is not achievable — **this is the top risk (see Antipattern watch / summary).**
  - **The playground-proxy allowlist extended, schema-driven from the in-repo OpenAPI**, to cover the subscription-creation path set (`/plans`, `/prices`, `/customers`, `/payment-methods`, `/subscriptions`) **and** the `/v1/test/*` instruments (`/v1/test/payment-methods`, `/v1/test/webhooks/simulate`), behind the test-base-only guard. Today the allowlist is only `/wallets`, `/payments`, `/payouts`, `/health` (`apps/docs/src/app/api/playground/route.ts`).
  - **The buffet snippet engine** (openapi.json + per-operation example registry → curl / Node / Python / Go / PHP / Ruby idiomatic snippets, rendered in `<CodeGroup>` with `localStorage` language memory). The grid and the tutorial consume it; they do **not** hand-write per-language strings.
- **Phase 01 — accent fix + shell.** The emerald accent (`--accent #0bdfa3` dark / `#00a473` light) must have replaced the stale purple ramp; the existing hero (`quickstart.tsx`), `steps.tsx`, and `card.tsx` still hard-code `purple-*` classes — this phase **reuses the corrected `primary`/`ring`/`border` tokens**, it does not re-fix them.
- **Phase 02 — manifest + Vale.** `content/manifest.ts` grouping conventions, the pager/breadcrumbs, and the Vale vocabulary pack (`:id` not `:ref`, `organizations` not `tenants`, integer `kobo`, sentence-case headings, second person, present tense).
- **Consumed from adjacent phases (cross-reference, do not rebuild):** Phase 05 owns the deterministic **Nigerian test-method registry** (`test_success`, `test_requires_otp`, `test_decline_insufficient_funds`, …) and the `/v1/test/webhooks/simulate` plumbing — this phase's tutorial and verify-step *use* those tokens and that call. Phase 08 owns the webhook **event catalog** and per-operation reference pages that these tutorials *link out to*. Phase 04 owns `/errors`, to which any error token surfaced here auto-links.

## Acceptance criteria (testable)

- **The 10-minute test.** A cold, anonymous reader lands, opens **Your first subscription** (or any grid stack page), clicks **Get a test key**, runs the steps, and sees a real `201` in devtools **in under 10 minutes with no signup**. Measured on a stopwatch, in at least Node + curl + one more stack.
- **Buffet reality.** The grid offers Node, Next.js, Python, Go, PHP, Ruby, CLI, curl; each stack page reaches `201` in its own language with **engine-generated** (not hand-typed) snippets, verified against the spec.
- **Honest, not mocked.** Every runnable in the section hits the real sandbox via the proxy with a test key; a live key is rejected `422`; devtools show real `X-Request-Id`s; the verify-webhooks event is a genuine signed payload that passes in-browser verification.
- **Diátaxis discipline.** No page in Get started contains an endpoint/field **reference table**; each such need is a link to Phase 08. Every page is a Tutorial (learning-oriented), launch-first.
- **No 404s.** Every link, CTA, card, pager step, and the starter-repo pointer resolves (the fixed hero CTAs included); the WebContainer (if shipped) links to the verify page for the inbound half.
- **Gates green.** Phase 03 OpenAPI honesty gate + no-404 check pass with these pages included; Vale passes; a11y AA holds on dark; Pagefind finds the new pages.
- **Money is integer kobo** everywhere in the section, with the naira-trap callout at the first amount.

## Antipattern watch (the §10 rules most relevant here)

- ❌ **A single forced tutorial path in a stack the reader doesn't use.** The grid is choice-first; a monolithic "getting started" silently says "we didn't build this for your stack."
- ❌ **Faked/`setTimeout` simulations or mocked `201`s.** A payments audience opens devtools; the tutorial and verify step must hit the real sandbox/instruments or trust collapses.
- ❌ **Live keys anywhere.** The ephemeral key and every explorer are `nbo_test_`-only; the proxy rejects live keys; simulate is test-deployment-only.
- ❌ **Calling the sandbox directly from the browser / breaking the `/v1/test/*` guard.** All runnable calls go through `/api/playground`; the verify step must not forward simulate to a live base.
- ❌ **Hand-maintained multi-language snippets.** Grid + tutorial + auth snippets come from the buffet engine, or they drift and lie.
- ❌ **Blending Diátaxis modes** — no reference tables in these tutorials; link to Phase 08 instead.
- ❌ **Dead links / vaporware pointers.** The starter-repo link is gated on the repo existing; the WebContainer must not claim to receive webhooks.
- ❌ **Happy-path-only onboarding.** The verify step includes `invoice.action_required`; the tutorial's collapsed sections point at the thin-balance/OTP hard parts (07), not hidden.
- ❌ **Marketing voice / Title Case / passive / future tense** on the front door — lead with the command.

## Manifesto ties

- **Tenet 2 — the developer is our first user.** Zero-signup ephemeral key, examples pre-filled, the fastest path to a real `201`; the section answers "what do I feel at 2am?" with relief.
- **Tenet 3 — a buffet, not a menu.** Eight stacks, each a runnable path; the answer to "can I use this with X?" is visibly yes, in X.
- **Tenet 4 — curiosity → first success in minutes.** The `<10-minute` target on every path; "verify us in your own devtools" makes the promise literal.
- **Tenet 6 — docs are the demo.** Every step is a real call; typed frontmatter keeps the section agent-readable for Phase 09.
- **Tenet 8 — honest about the hard parts.** The verify step is upfront that the sandbox sends no organic webhooks (simulate is the honest substitute); the WebContainer is upfront it can't receive them; the tutorial links the thin-balance/OTP realities rather than hiding them.
