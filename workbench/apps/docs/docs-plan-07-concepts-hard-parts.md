# Docs — Plan 07 · Concepts + the honest Hard Parts track (the honesty moat)

> Tell the truth about how money actually moves here, each hard truth paired with a runnable proof. Prerequisites: 04 (`/errors#CODE` + auto-link), 05 (`<LifecycleSimulator>`, `<EventInspector>`, `<PlaceYourBets>`, real lifecycle/money-flow presets, the `/v1/test/*` proxy + SSE + ephemeral key, the Nigerian test-method registry). Serves tenets: 8 (honest about the hard parts — honesty is the moat), 1 (the money is never wrong), 9 (errors are a feature), 6 (docs are the demo), 4 (curiosity → watch it happen).

## Goal

Build the **Concepts** section — Diátaxis **explanation** (understanding-oriented, plain Fly.io voice, never step-by-step) — and inside it the **Hard Parts** track: the repo's already-written honest essays imported and each paired with a runnable proof against the real sandbox. Concretely:

- **(a)** Import the hard-parts essays from `apps/website/content/hard-parts/*.mdx` into `apps/docs/content/concepts/hard-parts/*`, wire them into `content/manifest.ts`, and **honor each essay's `simulator:` frontmatter** by embedding the matching runnable proof at its exact failure branch (`insufficient_funds` → `<LifecycleSimulator scenario="thin-balance-recovery">`, `expired_card` → the card-update branch, `mid_run_interrupt` → `<IdempotencyLab>`, `transfer` → `<ReconciliationReplay>`, `mandate` → `<LifecycleStateMachine resource="mandate">`).
- **(b)** The **kobo money-unit** concept page + a `<MoneyUnit>` inline linter widget (type `₦5,000` → `500000` kobo → the exact JSON body; flag 100×-off values), with every amount field in the docs linking to it.
- **(c)** The `<IdempotencyLab>` double-charge explorable (**Pay** vs **Pay again with the same key** → watch one ledger movement; `IDEMPOTENCY_KEY_REUSED` on body mismatch).
- **(d)** The **card-recharge OTP/3DS wall** page — our live-confirmed finding (tokenized recharge forces OTP, ~0/2 headless) + the fallback we built (best-effort recharge → `invoice.action_required` → dunning-to-checkout-link), with a deterministic reproduce button.
- **(e)** The **reconciliation cookbook** (one late/double/dropped webhook → one correct balance) with an **adversarial replay** simulator (`<ReconciliationReplay>`: fire twice, out-of-order, drop+re-query → prove convergence).
- **(f)** Reconciliation **scrollytelling** (`<ReconciliationScrolly>`: a kobo's journey customer → ledger → escrow → sub-account, **every** step linking to a runnable endpoint — never a dead animation).
- **(g)** The **subscription-lifecycle** explanation + a **rail overview** (push/pull asymmetry).

Concepts explain; they do not instruct. Every task-shaped "how do I…" belongs to Guides (Phase 08) or Get started (Phase 06) — this phase cross-references those by number and never duplicates them.

## Antipattern watch (§10, most relevant here)

- ❌ **Faked simulations / try-its that hit a mock.** `<IdempotencyLab>`, `<ReconciliationReplay>`, and every embedded `<LifecycleSimulator>`/`<ApiExplorer>` hit the **real** sandbox via the Phase-03 proxy + `/v1/test/*`. A payments audience opens devtools; a `setTimeout` fake collapses trust.
- ❌ **Calling the sandbox directly from the browser** / **breaking the `/v1/test/*` test-base guard.** All traffic goes through the same-origin proxy/SSE; a live key is refused in-UI (defence in depth).
- ❌ **Scrollytelling that is pretty but not runnable.** Every `<ReconciliationScrolly>` step MUST terminate in a runnable endpoint or copy-run example — a dead animation ages into stale fiction (§10).
- ❌ **Happy-path-only.** The OTP wall, the double-charge trap, dropped/duplicate webhooks, and `churned` get equal or greater billing than recovery; the money-unit page leads with the 100× danger.
- ❌ **Error hints pointing at a 404 / presets frozen on `example`.** Every referenced code auto-links to a live `/errors#CODE`; the lifecycle/money-flow islands use the real subscription/reconciliation presets, never the deletable `example`.
- ❌ **Blending Diátaxis modes.** Concepts explain; no step-by-step, no reference tables masquerading as understanding — task content stays in Guides (08) / Get started (06), cross-referenced by number.
- ❌ **One concept, two names / marketing voice.** Vale enforces the locked vocabulary (organizations, `:id`, integer kobo); plain Fly.io explanation voice, no hype.
- ❌ **Dark mode without contrast math / purple drift.** New islands use emerald `--accent` + semantic tokens (danger `#f14949`, warning `#f6b84d`, success `#58cd78`) and meet AA on `#040404`; never the stale purple.
