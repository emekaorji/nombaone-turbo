# Docs — Plan 09 · Agent-native, changelog, merchant track, polish & launch
> Make the docs machine-readable and agent-callable, honest over time, merchant-inclusive, and launch-ready against §8. Prerequisites: **all** of 01–08 (hard: 02 voice/Vale + typed-frontmatter contract; 03 proxy `/v1/test/*` + SSE + snippet engine + ephemeral key; 04 `/errors`; 05 simulator verbs; 06 quickstart grid; 08 full reference + webhooks). Serves tenets: 3, 5, 6, 7, 8, 10 (with 1, 2, 9 on the launch gates).

## Goal

Close the 0→100 loop. Turn the docs from a human-only site into **docs-as-data**: every page has a clean Markdown mirror at a predictable URL with typed frontmatter, `/llms.txt` + `/llms-full.txt` are generated from `openapi.json` + the content tree and kept in CI lockstep, and a **copy-page-as-markdown / open-in-Claude** affordance sits on every page. Ship a **docs MCP server** and an in-docs **"Ask AI"** widget grounded *strictly* in the docs corpus + `/v1/openapi.json` + the error registry (never ungrounded — generated calls use correct field names and integer kobo). Upgrade **⌘K to execute, not just navigate** (run an endpoint, advance the test clock, simulate a webhook, trigger an error), sharing the console's verbs (tenet 10). Ship the **changelog** (diff per-release `openapi.json` snapshots → a Keep-a-Changelog MDX collection + RSS/Atom; defer any version-switcher), the no-code **merchant track** (named by what people control, one design system + voice), and a **"Coming from Paystack/Flutterwave"** migration guide. Finally, stand up the **launch gates**: the Vale pack in CI, a WCAG-2.2-AA audit on the dark default, CI-execution of every code sample against the sandbox *with a kobo-unit assertion*, the docs-gap feedback dashboard, and a final launch checklist mapped 1:1 to §8.

This phase adds **no new backend to `apps/api`** and touches `packages/*` only to **read** (`buildOpenApiDocument`, `ERROR_CODE_META`) at build time (§12). All runnable surfaces reuse the Phase 03 proxy — test keys only, real sandbox only.

## Prerequisites (what must exist first)

- **Phase 02** — `content/manifest.ts` as nav SSOT; the Vale rule-pack + locked vocabulary authored under `apps/docs/styles/`; the house voice; the agreed **typed-frontmatter contract** (this phase enforces it). The glossary.
- **Phase 03** — the playground proxy (`apps/docs/src/app/api/playground/route.ts`) extended to the real module allowlist + `/v1/test/*` (test-base-only guard) **and an SSE variant**; the **buffet snippet engine** + per-operation example-values registry; the ephemeral sandbox-key route; `buildOpenApiDocument` consumed in-repo at build; the error-code auto-link rehype transformer.
- **Phase 04** — `/errors` pages generated from `ERROR_CODE_META` (grounding target for MCP/Ask-AI and the migration codemods; every `docUrl` resolves).
- **Phase 05** — `<LifecycleSimulator>` + the test-toolkit verbs (advance-cycle, webhook simulate, deterministic test methods) — the actions ⌘K and Ask-AI drive.
- **Phase 06 / 08** — the quickstart grid and the full per-operation reference + webhook catalog (the corpus `llms-full.txt`, the changelog, and sample-execution index).
- Repo facts this phase builds on: `apps/docs/src/lib/content.ts` (`Frontmatter`, `getPage`, `listAllSlugs`); `apps/docs/content/manifest.ts`; `apps/docs/src/components/chrome/search-palette.tsx` (MiniSearch ⌘K, navigate-only today); `apps/docs/src/app/api/feedback/route.ts` + `src/lib/feedback.ts`; `apps/docs/scripts/build-search-index.ts`; `packages/errors/src/codes.ts` (`ERROR_CODE_META`, `PUBLIC_ERROR_CODES`, `DOCS_ERRORS_BASE`, `errorMetaFor`); `apps/api/src/shared/openapi/build.ts` (`buildOpenApiDocument`); `apps/website/content/hard-parts/moving-off-paystack-subscriptions.mdx` (Phase 07 import — cross-link, don't duplicate).

## Deliverables checklist

### (a) Docs-as-data — Markdown mirror, typed frontmatter, llms.txt

- [ ] **Formalize the typed-frontmatter schema.** Add `apps/docs/src/lib/frontmatter-schema.ts` exporting a Zod schema `frontmatterSchema` with `title: string`, `type: 'tutorial' | 'how-to' | 'reference' | 'explanation'`, `summary: string` (≤ 240 chars, one sentence), `prerequisites: string[]` (slugs, validated against `ALL_SLUGS`), `related: string[]` (slugs), `install?: string` (a single shell command), plus the existing optional `description`, `section`, `order`, `badge`. **Acceptance:** the schema is the single source; `type` is required on every content page and matches the page's Diátaxis section from the manifest.
- [ ] **Extend the content layer to carry the new fields.** Edit `apps/docs/src/lib/content.ts`: widen `interface Frontmatter` with `type`, `summary`, `prerequisites`, `related`, `install`, and parse them in `getPage` via `frontmatterSchema.safeParse` (no silent drops). **Acceptance:** `getPage('/getting-started/quickstart')` returns a fully typed `frontmatter.type === 'tutorial'` and `summary`.
- [ ] **Ship a build-time frontmatter validator.** Add `apps/docs/scripts/validate-frontmatter.ts` that walks `listAllSlugs()`, `safeParse`s each page's frontmatter, and **exits non-zero** listing every offending file+field. Wire it as a peer of `type-check` in the docs `turbo`/`package.json` (a new `docs:frontmatter` script). **Acceptance:** removing `summary` from any `.mdx` fails the build with a precise message; CI is red on drift.
- [ ] **Author the Markdown-mirror transform.** Add `apps/docs/src/lib/md-mirror.ts` exporting `pageToMarkdown(page: DocPage): string` — emits a clean, LLM-friendly Markdown doc: a typed YAML frontmatter block (title, type, summary, prerequisites, related, install, canonical URL) followed by the MDX body with each interactive island reduced to a one-line descriptor (e.g. `> Interactive: <LifecycleSimulator scenario="thin-balance"> — run it live at <url>`) and all `<CodeGroup>`/curl snippets flattened to fenced blocks. **Acceptance:** the output is valid CommonMark, contains no raw JSX, and every code sample survives verbatim.
- [ ] **Serve the mirror at a predictable URL (`<page>.md`).** Add `apps/docs/proxy.ts` (the Next.js 16 rename of `middleware.ts`) rewriting any request whose pathname ends in `.md` to `apps/docs/src/app/api/md/route.ts?slug=<path>`; the route calls `getPage` + `pageToMarkdown` and returns `text/markdown; charset=utf-8` (404 → a Markdown 404, never HTML). **Acceptance:** `curl https://docs.nombaone.xyz/getting-started/quickstart.md` returns clean Markdown for the same page the HTML route renders; `/does-not-exist.md` returns a Markdown 404, not the Next HTML 404.
- [ ] **Generate `/llms.txt` (curated index).** Add `apps/docs/scripts/build-llms.ts` (run inside `build`, before `next build`) that walks the manifest in nav order and writes `apps/docs/public/llms.txt` in the llms.txt convention: an `# Nomba One` H1, a blockquote one-paragraph summary, then one `##` block per manifest section with `- [Title](https://…/<slug>.md): summary` rows (summary from typed frontmatter). **Acceptance:** every manifest slug appears exactly once, links point at the `.md` mirror, and the file is regenerated deterministically.
- [ ] **Generate `/llms-full.txt` (inlined corpus).** In the same script, import `buildOpenApiDocument` from `@nombaone/api` (in-repo, no network) and write `apps/docs/public/llms-full.txt` = the concatenated Markdown of every page (`pageToMarkdown`) plus a compact rendering of each OpenAPI operation (method, path, summary, params, one example) and each `ERROR_CODE_META` entry (code, status, hint). **Acceptance:** an agent fed only `llms-full.txt` can name the correct endpoint, field, unit (integer kobo), and error code for "start a subscription that fails on a thin balance."
- [ ] **Add the llms lockstep CI check.** Extend `build-llms.ts` (or a sibling `apps/docs/scripts/check-llms.ts`) with a `--check` mode that regenerates in memory and diffs against the committed `public/llms.txt` / `public/llms-full.txt`, failing if they drift. Wire into CI. **Acceptance:** adding a page without regenerating fails CI; the agent view can never lag the human view (§4 agent-readable).
- [ ] **Ship the copy-page / open-in-Claude affordance.** Add `apps/docs/src/components/chrome/copy-page.tsx` (`'use client'` leaf) — a split button in the article header offering: *Copy as Markdown* (fetches `<slug>.md`), *Copy page URL*, *View as Markdown* (opens `.md`), *Open in Claude* (`https://claude.ai/new?q=` with a prompt embedding the `.md` URL), *Open in ChatGPT*. Mount it in `apps/docs/src/app/[[...slug]]/page.tsx` beside the breadcrumbs. **Acceptance:** one click yields the exact `.md` mirror on the clipboard; the button meets the a11y gate (labelled, keyboard-operable, 3:1 focus ring in `--ring` emerald on `#040404`).

## Acceptance criteria (testable)

1. **Docs-as-data:** every page resolves at `<url>.md` as clean CommonMark with typed frontmatter; `validate-frontmatter` fails the build on a missing `type`/`summary`; `/llms.txt` and `/llms-full.txt` regenerate deterministically and the `--check` mode fails CI on drift.
2. **Grounded agents:** the MCP server answers with real endpoints/fields/integer-kobo citing source URLs; Ask-AI refuses ungrounded questions and never emits a non-kobo amount (kobo linter runs on generated code); pasting an error envelope routes to the right `/errors` entry.
3. **⌘K executes:** typing an action verb runs it through the test-keyed proxy (SSE for multi-step), shows a real `X-Request-Id`, and rejects live keys before any call; verbs are byte-identical to the console's.
4. **Changelog:** a spec change produces a dated Keep-a-Changelog entry with deep-links + migration notes and a valid Atom feed; no version-switcher ships.
5. **Merchant + migration:** a non-engineer completes plan + payout-link setup with zero code; the migration guides map every Paystack/Flutterwave gotcha to a resolving target, ship a parallel-run playbook, and a CI-executed signature codemod.
6. **Launch gates:** Vale passes over the whole tree; the axe audit reports zero AA violations on dark; every code sample is CI-executed against the sandbox with a passing kobo assertion and a "last verified" badge; the docs-gap dashboard records zero-result searches + playground errors privately; `LAUNCH-CHECKLIST.md` maps 1:1 to §8 and is fully green; the link sweep finds no 404s and every `docUrl` resolves.

## Antipattern watch (the §10 rules most relevant here)

- ❌ **Ungrounded "Ask AI"** — the assistant must answer only from the retrieved docs + spec + error registry, cite sources, refuse when unsure, and never invent an endpoint, field, or (fatally) a naira amount. Grounding is the honesty gate; run the kobo linter on generated code.
- ❌ **AI/⌘K as decoration** — the palette must *execute* real test-instrument actions and the widget must give grounded answers; if it can only fuzzy-match titles, don't ship the affordance.
- ❌ **Calling the sandbox directly / breaking the test guard** — every executable action and the sample verifier go through the Phase 03 proxy; test keys only; `/v1/test/*` never forwards to a live base; live keys rejected before any call.
- ❌ **A version-switcher before v2** — keep the changelog spec-diff-driven; ship no version dropdown.
- ❌ **Happy-path-only / near-synonym IDs** — the migration guide teaches one canonical `reference` (not the `tx_ref`/`flw_ref`/`trxref` tangle) and links the honest reconciliation/OTP-wall pages, never a 404.
- ❌ **Blending Diátaxis modes / two voices** — the merchant track is named by what people control and cross-links (not embeds) developer detail; Vale enforces one voice across the whole tree.
- ❌ **Indexing stripped markdown / hand-maintained agent view** — `llms.txt` + the `.md` mirrors are generated and CI-locked to the content + spec; the dashboard learns from real zero-result searches (self-correcting docs).
- ❌ **Dark mode without contrast math** — the axe audit is a hard gate; no decorative low-contrast accent, no stray `purple-*` (use `--accent` emerald).

## Manifesto ties

- **Tenet 6 (docs are the demo / if it isn't in the docs it doesn't exist):** the `.md` mirror + `llms.txt`/`llms-full.txt` + MCP make the whole surface machine-consumable; the sample gate keeps it runnable-and-honest.
- **Tenet 3 (a buffet, not a menu):** "can I use this from my agent?" becomes a literal yes via MCP + Ask-AI + the buffet-language migration codemods.
- **Tenet 5 (the API is the product):** the changelog and `llms-full.txt` derive from `buildOpenApiDocument`, so the agent/human/changelog views can't drift from the served spec.
- **Tenet 7 (merchants deserve the same care):** a first-class no-code track under one design system + voice, named by what people control.
- **Tenet 8 (honest about the hard parts):** the migration guide tells the truths Paystack/Flutterwave hide (unreliable webhooks → reconciliation; the ID tangle → one canonical reference); the changelog carries migration notes for every breaking change.
- **Tenet 10 (one platform, one language):** ⌘K shares the console's verbs; the Vale pack enforces one vocabulary across the whole tree; the glossary and MCP speak one dialect.
- **Tenets 1 / 2 / 9 (launch gates):** the kobo assertion in CI keeps the money never wrong; the migration/merchant tracks serve the developer's and merchant's 2am relief; the docs-gap dashboard turns every dead-end error/search into the next fix.
